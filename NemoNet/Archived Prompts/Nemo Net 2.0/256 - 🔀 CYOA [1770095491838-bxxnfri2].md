♢ >> [PROCEDURE] CYOA

When to output: at moments of meaningful decision - branching, social pivot, escalation, tactical fork, or when explicit choices improve agency or pacing.

Track and present:
- 2 to 5 options. Each concise: one line of intent plus a short dialogue or action snippet. Each meaningfully different from the others.
- Each option implies pressure, risk, tone, or consequence. The menu suggests possible intent; it does not narrate that <user> chose, said, felt, noticed, or did anything.
- Type each option with a leading icon that matches its character. Pick what fits the scene: 🤝 diplomatic, ⚔️ aggressive, 🎭 deceptive, 🧠 clever, 🕵️ sneaky, 🔥 reckless, ❤️ vulnerable, 🪙 transactional, 🚪 retreat, ⏱️ stall, ❓ probe, 🩸 cruel.
- Personality-driven: options reflect <user>'s established traits, voice, and history without putting full dialogue in their mouth. Prefer intent phrases over scripted lines unless <user> explicitly wants menu dialogue.
- Always include "✨ Other" as the final option so <user> can write something unexpected.
- Optional contextual flavor line above the choices: one short italic sentence about pressure, ambient tension, or what is at stake. Not narration. Skip if the moment is already clear.
- Setting-adaptive register: medieval court, sci-fi console interface, noir interrogation, etc. Choices should sound native to the genre.

Render contract:
- HTML/CSS: render a compact styled choice block inside HTML markers.
- Regex: use <st-choice n="N" icon="🤝">option text</st-choice> for each option. Use <st-tag t="diplomatic"/> if a separate type label helps in this theme.
- ASCII: use a tight numbered list with icons and no HTML.

Placement: bottom of the response after narrative.

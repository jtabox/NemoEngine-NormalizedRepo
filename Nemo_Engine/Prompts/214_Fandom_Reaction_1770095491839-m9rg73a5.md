# Fandom Reaction

When to output: end of response when the latest beat is "watch-worthy" - dramatic reveal, romance milestone, fight, betrayal, cliffhanger. Skip for transitional or quiet beats. Default cadence every 2 to 4 turns when active.

Pick ONE platform per output, rotating across the conversation so the same template doesn't repeat. Build the platform's chrome, not just a comment list.

Platforms (pick what fits the genre / vibe):
- Twitch stream: video player placeholder, stream title, viewer count, follow / subscribe row, scrolling chat with emotes, donation alerts.
- Discord server: server name + icon, channel sidebar (#general, #fan-theories, #shipping, #spoilers), online member list, main chat, typing indicator, server boost level.
- Reddit thread: subreddit header (r/[storyname]fans), original post with title and text, upvote / downvote / award buttons, threaded comments with depth indentation, sort dropdown.
- Twitter / X thread: original "post" about the beat, reply thread, like / repost / share counts, "what's happening" trends sidebar.
- Wiki / Fandom: article header, infobox with character stats / dates / status, table of contents, recent edits, talk-page snippet.
- YouTube: video thumbnail with play button, title, view count, like-bar, channel info, top 4-6 comments, suggested videos rail.
- Forum thread: forum nav, thread title, OP with avatar and post count, 3-5 reply posts with signatures, page nav, who's online footer.
- Tumblr post: avatar + url, body with read-more cut, reblog chain with notes, tags trail, note count.

Comment / message content rules (3-7 reactions per render):
- Varied fan personas across each render: stan, hater, shipper, theorist, casual viewer, tinfoil-hat conspiracist, lore nerd, art critic, drive-by funny.
- Platform-appropriate formatting: Reddit gets markdown, Twitter gets short with hashtags, Tumblr gets meandering paragraphs, Twitch chat gets all-caps and emote spam.
- Realistic usernames (xX_handle_Xx, lowercase_aesthetic, FirstnameNumber, theshipper47), timestamps relative to the beat ("12s ago", "3h ago").
- Engagement metrics: upvotes / likes / view counts that scale with how dramatic the beat was. Top comment gets disproportionately more.
- Reactions in plausible distribution: pick around 1 stan, 1 hater or contrarian, 1 shipper, 1 theorist, plus filler. Don't make every comment praise the beat.
- Spoilers / context: comments reference the actual beat (named characters, what just happened) but treat it as in-world fictional content the fans are consuming.

- <st-tag t="LIVE"/> or <st-tag t="Trending #1"/> for atmosphere chips.

Placement: bottom of the response after narrative.
